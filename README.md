# Father List Bot 
Telegram-бот для керування публікаціями, каналами та базовою автоматизацією розсилки. Проєкт побудований на **Python**, **aiogram** та **SQLite** і орієнтований на роботу з Telegram-каналами, чатами, ролями користувачів та шаблонами постів.

## Що вміє бот

- реєструвати користувачів у локальній базі даних;
- розділяти ролі на **адміністратора**, **модератора** та **звичайного користувача**;
- показувати різні меню залежно від ролі;
- додавати Telegram-канали та групи для подальшої публікації;
- перевіряти, чи має бот потрібні права в каналі;
- створювати текст рекламного або інформаційного поста з кількох частин:
  - верхня частина;
  - середина;
  - нижня частина;
- зберігати посилання на медіа;
- перемикати тип розсилки: **з медіа** або **без медіа**;
- виконувати базовий автопостинг із папки з файлами у Telegram-канал;
- вести просту локальну базу даних користувачів, каналів і службових даних.

## Основна ідея проєкту

`Father List Bot` — це керуючий Telegram-бот, який поєднує в собі:

1. **Адмін-панель у Telegram** для керування функціями.
2. **Роботу з каналами та чатами** через додавання їх у систему.
3. **Збереження шаблонів постів** у SQLite.
4. **Автопублікацію** контенту з локальної папки у вибраний канал.
5. **Поділ доступу** між адміністратором, модератором і звичайним користувачем.

Такий підхід зручний для невеликого SaaS-бота, сервісу автопублікації або внутрішнього інструмента для керування Telegram-мережею.

## Структура проєкту

```text
Father_Listbot/
├── admins.py                # Логіка ролей адміністратора та модератора
├── config.py                # Токени, ID, канал, шлях до медіа, інтервал
├── db.py                    # Робота з SQLite базою даних
├── first_bot.py             # Спрощений окремий приклад автопостингу
├── format_text.py           # Тексти меню, довідки, описів, повідомлень
├── func.py                  # Додаткова бізнес-логіка бота
├── main.py                  # Основна точка входу
├── markups.py               # Reply / Inline клавіатури
├── media.py                 # Збереження медіа-посилання та типу розсилки
├── text_castomise.py        # Генерація форматованих посилань у тексті
├── text_loader.py           # FSM-стани для введення текстів / даних
├── requirements.txt         # Залежності проєкту
├── user_db.db               # Локальна база SQLite
└── starter Father_listBot.bat  # Швидкий запуск у Windows
```

## Як працює бот

### 1. Запуск
Основний файл запуску — `main.py`.

Під час старту бот:
- підключає залежності;
- створює підключення до бази даних `user_db.db`;
- завантажує меню та тексти;
- реєструє хендлери повідомлень і callback-кнопок.

### 2. Робота з користувачами
Користувач після `/start`:
- перевіряється в базі;
- за потреби додається в БД;
- отримує меню відповідно до своєї ролі.

### 3. Робота з ролями
У проєкті реалізовані ролі:
- **admin** — повний доступ до функцій;
- **moder** — обмежений доступ до частини керування;
- **user** — базове користування та інформаційні розділи.

### 4. Робота з каналами
Адміністратор або модератор може:
- додати канал або чат;
- переслати повідомлення з каналу для отримання його ID;
- перевірити, чи має бот права адміністратора;
- зберегти канал у базі даних.

### 5. Робота з постами
Пост збирається з кількох частин:
- шапка;
- основний текст;
- нижній блок.

Ці частини зберігаються в БД і можуть формувати фінальний текст для публікації.

### 6. Автопостинг
У проєкті є базова логіка автоматичної публікації:
- бот бере випадковий файл із папки;
- визначає його тип;
- надсилає фото або документ у канал;
- працює за вказаним інтервалом у хвилинах.

## Встановлення

### 1. Клонувати репозиторій
```bash
git clone https://github.com/your-username/Father_Listbot.git
cd Father_Listbot
```

### 2. Створити віртуальне середовище
```bash
python -m venv .venv
```

### 3. Активувати середовище
**Windows:**
```bash
.venv\Scripts\activate
```

**Linux / macOS:**
```bash
source .venv/bin/activate
```

### 4. Встановити залежності
```bash
pip install -r requirements.txt
```

## Налаштування

Перед запуском потрібно відредагувати `config.py`.

Приклад логічних параметрів:

```python
# Токен Telegram-бота
TOKEN = "YOUR_BOT_TOKEN"

# ID адміністратора
admin_id = 123456789

# Канал для автопостингу
CHANNEL_ID = "@your_channel"

# Папка з медіафайлами
FOLDER_PATH = "C:/path/to/media"

# Інтервал автопублікації у хвилинах
interval = 60
```

## Запуск

```bash
python main.py
```

Або у Windows можна використати `.bat`-файл запуску.
